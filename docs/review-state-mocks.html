Review Button
Approval is keyed by branch surface hash. Clean commit approval is `commit`; dirty approval is `commit + working-tree hash`. A previously approved surface restores automatically.
commit approved + clean Approved
commit unapproved + any dirty state Approve
commit approved + dirty hash unapproved Approve working tree
commit approved + dirty hash approved Approved

Switch to update-review for commit drift. If HEAD matches the generated commit but the branch hash changed, show “working tree changed” instead of “commit behind”.
Rule of thumb: pending drift protects the reader from sudden jumps while the page is open. Reload converts pending facts into loaded facts. Once loaded, dirty state must be visible persistently and approval must be freshness-based, not just SHA-based.
