Option 3 — compare-primary, developed

The compare branch is the subject; base is a quiet reference; the worktree is a decoupled "edit target." · local-review

The shape: the compare branch is the prominent chip — it's what you're reviewing. The base is a small affordance beside it. Both open the same picker (filter → branches → commits → Apply). The worktree leaves the ref pickers entirely and becomes a separate edit target: it only matters when you apply edits, never when you view. This page answers the two open questions — where the worktree goes and how the base affordance looks.

A Where the worktree goes

Decoupled from viewing. The worktree only answers one question now — "where do this branch's edits land?" — so it becomes a quiet edit-target control. Because the branch is already fixed (it's the compare ref), the worktree popover loses its branch row entirely: that's what removes the "worktree jumps when I pick a branch" friction. Three placements, recommended first.

RecommendedA1 · Edit-target chip on the header rightgrouped with workspace concerns (session / chat), away from the ref chips

Left of the header = what am I comparing (refs). Right of the header = working context (which worktree edits go to, which session, chat). Always visible but quiet; it goes amber only when it's actually blocking you.

B How the base affordance looks

Base is secondary, but it must (1) read clearly as the thing you compare against, (2) open the same picker as compare, and (3) not undo your recent base-left ordering work. Two trigger treatments, then the shared popover both sides use.

RecommendedB1 · Base-left, compare-prominentkeeps your "base ← compare" order; prominence comes from styling, not position

Reconciles compare-primary with the base-left commit you just landed: main stays on the left as quiet text, then the arrow, then the bold compare chip. You read it left-to-right as "main → feature," and the heavy chip still tells you which one is the subject.

Compare comes first, base trails as vs main. Most natural reading of "I'm looking at this branch vs main," but it does flip the base-left order from your recent commit.
